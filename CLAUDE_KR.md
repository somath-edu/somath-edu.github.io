# Archive S - AI 가이드라인 (CLAUDE.md)

이 문서는 Archive S 디지털 도서관 프로젝트를 수행하는 AI 어시스턴트를 위한 지침입니다. 모든 AI 에이전트는 프로젝트의 무결성과 디자인 철학을 유지하기 위해 이 규칙을 최우선으로 준수해야 합니다.

## 1. 핵심 사고 프로세스 (엔트로피 기반 추론)
- **엔트로피 증가 (확산)**: 작업 지시를 받으면 즉시 실행하지 않는다. 먼저 관련된 모든 정보, 예외 상황, 가능성을 수집하고 분석한다.
- **엔트로피 감소 (구조화)**: 확산된 정보를 논리적이고 단순한 구조로 정리한다. 복잡한 혼돈 상태를 명확한 계획으로 압축한다.
- **우선순위 기반 실행**: 구조화된 계획을 바탕으로 우선순위 목록을 작성하고, 최상위 항목부터 하나씩 순차적으로 실행한다.
- **체계적 보고**: 작업 과정에서 현재 작업의 하위 구조와 전체 프로젝트 계층 구조에서의 위치를 명확히 설명한다.

## 2. UI/UX 디자인 철학
- **모바일 우선 & 엄지손가락 최적화**: 모바일 사용자를 최우선으로 디자인한다. 하단 내비게이션 바, 큰 터치 타겟, 가로 스와이프(카루셀) UI를 적극 활용한다.
- **프리미엄 디지털 도서관 미학**: 다음 요소들을 사용하여 고급스러운 '리더 앱' 느낌을 유지한다:
    - **다크 모드**: 딥 네이비/슬레이트 톤의 배경.
    - **글래스모피즘**: 배경 흐림(Blur) 효과가 적용된 반투명 표면.
    - **타이포그래피**: 제목에는 우아한 세리프체(Playfair Display), 본문에는 깔끔한 산세리프체(Pretendard) 사용.
- **반응형 설계**: 모바일에서 태블릿, 데스크탑으로 자연스럽게 확장되는 레이아웃을 보장한다.

## 3. 프로젝트 컨텍스트 및 기술 규칙
- **정체성**: 이 프로젝트는 단순한 블로그가 아니라, 지식을 연결하고 조직화하는 '디지털 도서관'이다.
- **수학 콘텐츠 최적화**: 수학 관련 콘텐츠(미적분 등)가 모바일에서도 완벽하게 읽히고 반응형으로 동작하도록 보장한다.
- **안전 프로토콜**: `index.html`이나 핵심 CSS 파일을 수정하기 전, 반드시 백업(예: `파일명_backup.html`)을 생성한다.
- **플레이스홀더 금지**: 일반적인 플레이스홀더를 사용하지 않는다. 실제와 같은 고품질 콘텐츠를 사용하거나 사실적인 에셋을 생성하여 적용한다.

## 4. 기술 스택

- **스타일링**: Vanilla CSS (`bookstore.css` 기반)
- **아이콘**: Lucide Icons (CDN 활용)
- **배포**: GitHub Pages
